The problem

Many diasporans find it challenging registering, renewing or recovering their Identity cards, namely passports, driver's licences, and so on.

Green I.D. is a company that wants to alleviate these challenges by providing an informative, and multi-service-driven platform that solves all of these problems.

Planning & Research

In order for me to build a multi-service platform, I had to understand how the services operated. with this in mind I set out to research the first business that will launch immediatedly the platform is done. This business is a requirement by te government for diasporans to register for their National Identity Number before they can go back or do anything within their country.

I first researched all the government requirements to register diasporans abroad, the type of companies that can meet these requirements and how they will make their money while offering this service. This led to clear understanding of the complete business process.

With this infomration in mind, I looked at the pattern of operation of three other services that will use the platform, Passport support service, Drivers licence support service, and bank verification number service.

KEY FINDINGS
All the companies process, follows a simillar pattern; You fill a form online or in office and then get a biometric capture.

Strategy, User Needs & Business Goals

My first time interacting with a stakeholder, I make sure I clearly understand their vision, budget, the stage they are in on the project, the amount of data gathered, and how valid it is, so I can figure out what UX strategy is required for the project. Then I find a place for the project in one of my four project categories, this helps to accelerate figuring out the right UX path to take for the project. The categories are MVP, OVP, and Data-driven.

DEFINITION
Minimum Viable Product: Testing a product hypothesis with little time or resources. Optimum viable product: The market is well established and understood, There is access to user data for testing. Data-driven product: Already established product with details to concrete data, and a short feature release cycle.

Deliverables

Open each section to see why I chose each task as part of my UX strategy.

UX Strategy Documentation Every UX project starts with a strategy, I develop a UX strategy based on data and requirements the company provides.

Design Thinking Documentation When on a project alone, I always start my projects with a one-hour design thinking exercise, this helps me do a mind dump on my assumptions on the project, figure out stakeholders I will need for the project, and do some exercises to figure out the desired outcome. When in a team, This exercise will take two hours with the same process.

Value Proposition I chose to analyse the value proposition to be able to understand the customer's pain virsus the solution.

User Research I chose to run user research to hear the user's needs, understand the actual needs, and highlight the unknown needs, which will enable me to advocate for the user.

Industry & Market Research I chose to run industry and market research to validate the data given to me, deeply understand the business, and be able to see how I can add value to the research at hand.

Competitive Analysis I chose to run a competitive analysis to compare research and validate, and understand the business, the pattern of operations, technology and trends.

Riskiest Assumptions I chose to run the riskiest assumptions research, to be able to identify, categorise, analyse, and validate assumptions.

IA & Site map I chose IA and sitemap because I am designing a digital platform that requires the provision of these data, to be able to figure out how users will navigate the platform.

Sketches, Wireframes, Prototypes I chose sketches, wireframes and prototypes because I was allocated the time required to run a full visual research and design.

User Testing I chose user testing to be able to start testing my model and solving usability problems at an early stage.

A/B Testing I chose A/B testing as part of the UX process because the application has some complex navigation problems that needed more than one design.

Fonts, Colours, Typography, Spacing I chose to provide these requirements because I am building a digital platform that requires an exciting user interface.

Style Guides, Pattern Library, Design Sytems I chose to provide these elements because the project will be evolving and will have large teams in the future.

Design I chose to design the application because it was part of the deliverables, and I have the skills to deliver.

High fidelity prototype I chose to provide a high-fidelity prototype because we needed to be able to test and use the prototype as part of the pitch deck.

Pattern Library Development I chose to develop the pattern library because it will be easier for collaboration than design, this reduces collaboration time by 50%.

Bootstrap Prototype I chose to create a bootstrap prototype because the project launch failed due to funding, with a bootstrap prototype it was easier to move the project to the development phase with a reduction in development time by 60%.
